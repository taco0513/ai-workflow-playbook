# Context Engineering Quick Reference (250토큰)

## 핵심 원칙 (50토큰)
- 정확한 정보 > 많은 정보
- 구조화된 입력 = 정확한 출력
- 컨텍스트 = AI의 작업 메모리

## 최적 패턴 (100토큰)
```
[상황] + [제약] + [목표] + [시도]

예시:
"Next.js 14에서 hydration 에러 발생
브라우저: Chrome, 프로덕션 환경만
목표: 사용자 영향 없이 수정
시도: useEffect 확인했지만 해결 안됨"
```

## 컨텍스트 타입 (50토큰)
- 🔧 기술: 버전, 환경, 스택
- 📊 데이터: 에러 로그, 메트릭
- 🎯 목표: 기한, 우선순위
- ⚠️ 제약: 예산, 기술, 시간

## 효율 팁 (50토큰)
✅ 에러 메시지 전체 복사
✅ 파일 경로 정확히 명시
✅ 이미 시도한 것 나열
❌ 불필요한 배경 설명
